# Документация проекта (Docs-as-Code с AsciiDoc → DOCX)

---

Подход **Docs-as-Code** — документация хранится как код и собирается автоматически.

Процесс выглядит так:

```
.adoc (текст) → HTML → DOCX (Word)
```

---

## Зависимости

Используются:

* **Asciidoctor (2.0.26)** — преобразует `.adoc` в HTML;
* **Pandoc (3.9.0.2)** — преобразует HTML в `.docx`;
* **Make (3.81)** — запускает сборку одной командой.

---

## Структура проекта

```
docs/.              # Корневая директория проекта
├── src/            # Исходные тексты для сборки документа
├── styles/         # Стили Word + логика оформления
├── scripts/        # Скрипт сборки
├── build/          # Результат в docx
├── Makefile        # Запуск
└── README.md       
```

## Установка зависимостей

### Linux (Ubuntu / Debian)

Откройте терминал:

```
sudo apt update
sudo apt install asciidoctor pandoc make
```

---

### macOS

```
brew install asciidoctor pandoc make
```

---

### Windows

Самый простой способ:

1. Установите **WSL (Linux внутри Windows)** (https://learn.microsoft.com/ru-ru/windows/wsl/install)
2. Откройте терминал Ubuntu
3. Выполните:

```
sudo apt update

```

## Сборка документа

Перейти в корневую директорию проекта:

```
cd docs
```

Запустить:

```
make build
```

---

## Просмотр результата

После сборки финальный документ будет находиться в директории:

```
build/docx/output.docx
```

## Написание документации

Все тексты находятся в папке:

```
src/
```

Главный файл:

```
src/index.adoc
```

Пример:

```
= Мой документ

Это обычный текст.

[.warning]
Это предупреждение.

[.note]
Это заметка.
```

## Добавление собственных стилей

Применеие стилей происходит в два этапа:

### 1. Настройка соответсвия роли `.adoc` и стиля `.docx` в файле `styles.lua`

> Некоторые роли (и, соответственно, стили в выходном документе) присвоить нельзя, например, для таблиц при генерации документа .docx автоматически применяются базовые стили Word, но если необходимые стили добавлены в файл reference, то в сгенерированном документе они будут отображаться в панели стилей и их можно применить вручную.

В `.adoc` текст можно пометить соответсвующей ролью, от которой будет зависеть стиль:

```adoc
[.warning]
Предупреждение

[.note]
Заметка

[.CustomText]
Кастомный текст

[.highlight]#подсветка#
```

Далее необходимо внести изменения в `styles.lua`, добавив обработку ролей:

```lua
    elseif class == "note" then
      el.attributes["custom-style"] = "NoteText"
```

В примере `"note"` - роль в `.adoc`, а `"NoteText"` - название стиля в `.docx`


### 2. Добавление стиля в `reference.docx`

Необходимо открыть файл:

```
styles/reference.docx
```

В Word:

1. Найти "Стили".
2. Изменить:

   * `WarningText`
   * `NoteText`
   * и другие

3. Добавить необходимые стили.

## Частые ошибки

### `make: command not found`

→ не установлен make

---

### `asciidoctor: command not found`

→ не установлен asciidoctor

---

### стили не применяются

Проверьте:

* имя стиля в Word совпадает с именем стиля в styles.lua
* роль написана правильно (i.g. `.warning`, а не `.wrning`)

---

### файл не создаётся

Проверьте:

```
scripts/build.sh – наличие скрипта в директории
```

и права:

```
chmod +x scripts/build.sh
```
