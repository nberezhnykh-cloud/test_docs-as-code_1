# 📚 Документация проекта (Docs-as-Code с AsciiDoc → DOCX)

Этот проект показывает, как из простых текстовых файлов (`.adoc`) автоматически собирать красиво оформленные документы Word (`.docx`).

Даже если вы никогда раньше не работали с такими инструментами — не переживайте. Ниже всё объяснено максимально просто.

---

# 🧠 Что здесь происходит

Мы используем подход **Docs-as-Code** — документация хранится как код и собирается автоматически.

Процесс выглядит так:

```
.adoc (текст) → HTML → DOCX (Word)
```

---

# 🧰 Какие инструменты используются

Вам понадобятся:

* **Asciidoctor** — преобразует `.adoc` в HTML
* **Pandoc** — преобразует HTML в `.docx`
* **Make** — запускает сборку одной командой

---

# 📁 Структура проекта

```
docs/
├── src/            # Исходные тексты (редактируете здесь)
├── styles/         # Стили Word + логика оформления
├── scripts/        # Скрипт сборки
├── build/          # Результат (генерируется автоматически)
├── Makefile        # Упрощённый запуск
└── README.md       # Эта инструкция
```

---

# ✍️ Где писать документацию

Все тексты находятся в папке:

```
src/
```

Главный файл:

```
src/index.adoc
```

Пример:

```
= Мой документ

Это обычный текст.

[.warning]
Это предупреждение.

[.note]
Это заметка.
```

---

# 🎨 Что такое роли (очень важно)

Роли — это способ сказать:

> "Этот текст — особенный"

Пример:

```
[.warning]
Внимание!
```

Это НЕ делает текст красным сразу.
Это просто помечает его как "warning".

Дальше система сама применит стиль в Word.

---

# 🎯 Как применяются стили

Происходит магия в два этапа:

### 1. `styles.lua`

Говорит:

```
.warning → стиль WarningText
```

### 2. `reference.docx`

Говорит:

```
WarningText = красный текст, жирный и т.д.
```

👉 То есть:

* `.adoc` = смысл
* `.docx` = внешний вид

# ⚙️ Установка (пошагово)

## 🐧 Linux (Ubuntu / Debian)

Откройте терминал:

```
sudo apt update
sudo apt install asciidoctor pandoc make
```

---

## 🍏 macOS

```
brew install asciidoctor pandoc make
```

---

## 🪟 Windows (рекомендуется)

Самый простой способ:

1. Установите **WSL (Linux внутри Windows)**
2. Откройте терминал Ubuntu
3. Выполните:

```
sudo apt update
sudo apt install asciidoctor pandoc make
```

---

# ▶️ Как собрать документ

Перейдите в папку проекта:

```
cd docs
```

Запустите:

```
make build
```

---

# 📦 Где результат

После сборки файл появится здесь:

```
build/docx/output.docx
```

Откройте его в Word 🎉

---

# 🔁 Как это работает внутри

Команда `make build` делает:

1. `.adoc → HTML`
2. HTML → `.docx`
3. Применяет стили

---

# 🧪 Пример ролей

Вы можете использовать:

```
[.warning]
Предупреждение

[.note]
Заметка

[.custom-paragraph]
Кастомный текст

[.highlight]#подсветка#
```

---

# 🎨 Как изменить внешний вид документа

Откройте файл:

```
styles/reference.docx
```

В Word:

1. Найдите "Стили"
2. Измените:

   * `WarningText`
   * `NoteText`
   * и другие

💡 Все документы автоматически обновят внешний вид

# 🚨 Частые ошибки

## ❌ `make: command not found`

→ не установлен make

---

## ❌ `asciidoctor: command not found`

→ не установлен asciidoctor

---

## ❌ стили не применяются

Проверьте:

* имя стиля в Word совпадает
* роли написаны правильно (`.warning`)

---

## ❌ файл не создаётся

Проверьте:

```
scripts/build.sh
```

и права:

```
chmod +x scripts/build.sh
```

---

# 💡 Советы

* Пишите просто — система всё оформит сама
* Используйте роли вместо ручного форматирования
* Не редактируйте файлы в папке `build/`
